# Legal Company Brain Quality Checklist

Use this checklist when reviewing answers generated from a legal company brain.

## 1. Source grounding

- Does every consequential claim point to a source?
- Are official sources and final approved internal materials preferred over informal context?
- Does the answer identify source gaps?
- Does it flag stale or conflicting sources?

## 2. Completeness

- Does the answer identify the relevant product, jurisdiction, data category, vendor, customer commitment or regulatory obligation?
- Does it distinguish binding commitments from informal statements?
- Does it identify the right internal stakeholders?
- Does it avoid presenting a partial source set as complete?

## 3. Judgment and uncertainty

- Are facts, assumptions and judgment separated?
- Are missing facts clearly stated?
- Does the answer state what would change the conclusion?
- Does it avoid overconfident language where the source base is weak?

## 4. Escalation quality

- Does the answer identify escalation triggers?
- Does it recommend an owner or next responsible function?
- Does it specify when qualified human review is required?
- Does it flag board, privacy, security, regulator-facing or external-specialist escalation where appropriate?

## 5. Confidentiality and access

- Does the answer avoid disclosing restricted context to unauthorised users?
- Does it treat personal data and regulated data carefully?
- Does it avoid using real client or customer matter data in public examples?
- Does it make access assumptions visible?

## 6. Playbook value

- Does the answer create a reusable pattern for future matters?
- Does it suggest a memory update if the answer revealed a gap?
- Does it identify stale decisions that should be reviewed?
- Does it improve the operating system, not merely answer one question?

## Scoring

| Category | Points |
|---|---:|
| Source grounding | 0 to 2 |
| Completeness | 0 to 2 |
| Judgment and uncertainty | 0 to 2 |
| Escalation quality | 0 to 2 |
| Confidentiality and access | 0 to 1 |
| Playbook value | 0 to 1 |
| **Total** | **0 to 10** |

## Automatic fail

The answer fails automatically if it:

- presents a high-impact conclusion without qualified human review
- misses a clear escalation trigger
- exposes restricted material to the wrong audience
- ignores a binding customer commitment
- claims certainty from incomplete or stale sources
